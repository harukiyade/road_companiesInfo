## 4. インポート用マッピング定義ドラフト

### 4.1 振り分けルール（データの「濃さ」に応じた最適化）

| 判定 | 条件 | 格納先 |
|------|------|--------|
| **詳細** | 文字数 50字以上、または 事業・財務・強み・沿革などを含む | **business_descriptions** |
| **補足的概要** | 1〜2行程度（おおよそ50字未満）、業種の端的説明 | **business_summary** |
| **備考・コメント** | 会社情報・備考・担当者コメント・コメント | **company_description** |

### 4.2 CSVヘッダー → DBカラム マッピング案

| CSVヘッダー | 第1候補 | 第2候補 | 備考 |
|-------------|---------|---------|------|
| **概要** | business_descriptions | - | 詳細な説明が多い |
| **説明** | business_summary | overview | 端的な1行説明 |
| **企業概要** | business_descriptions | business_summary | 内容により文字数で振り分け |
| **概況** | business_descriptions | business_summary | 同上 |
| **事業内容** | business_descriptions | - | 詳細情報 |
| **特意分野** | business_descriptions | - | 専門性の詳細 |
| **会社情報・備考** | company_description | - | 備考系 |
| **備考** / **担当者コメント** / **コメント** | company_description | - | 備考系 |

### 4.3 実装案（import_full_update_fast.py 用）

```
【オプションA】単純マッピング（ヘッダー単位）
- 説明 → business_summary
- 概要 → business_descriptions
- 企業概要 → business_descriptions（yuzuri等）
- 会社情報・備考, 備考, 担当者コメント, コメント → company_description
- overview は 説明 がなく 概要 のみの場合は 概要 の先頭50文字を summary 的にも格納可能

【オプションB】文字数ベースの振り分け
- 説明/概要/企業概要 の値を取得後、50字以上 → business_descriptions、50字未満 → business_summary
- 同一行に 説明 と 概要 が両方ある場合：説明→business_summary、概要→business_descriptions
```

### 4.4 推奨

**オプションA** を推奨。108.csv の分析から「説明＝短い」「概要＝詳細」が明確なため、ヘッダー単位の固定マッピングで十分。

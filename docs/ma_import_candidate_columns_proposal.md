# MAツール向け 追加取り込み候補カラムの提案

企業データベースの取り込み対象として**確定済み**のカラム以外で、ターゲティング・セグメント配信・企業スコアリング・トークスクリプト作成に有用なカラムを、活用シーンとともに提案する。

---

## 1. 業績・規模に関する情報

| カラム名 | 提案理由・活用シーン |
|----------|----------------------|
| **直近売上** | 企業規模でセグメント（大企業／中小／零細）。リードスコアの「購買力」要素。営業優先度づけ・アウトバウンドのターゲット選定に必須。 |
| **直近利益** | 収益性の可視化。黒字企業と赤字企業で配信メッセージやオファーを切り替え。スコアリングの「財務健全性」指標。 |
| **直近決算年月** | データの鮮度判定。決算期直後は予算・意思決定のタイミングとして営業アクションの最適化に利用。 |
| **売上規模（百万円）** | 直近売上と同様の規模セグメント用。単位が「百万円」で統一されたファイルでの規模軸。 |
| **社員数** | 従業員規模によるセグメント（例：50人未満／50〜300／300人以上）。組織サイズに応じた提案・案件サイズの見積もりに有用。 |

**MAでの典型的な使い方**: セグメント「直近売上○億円以上」「社員数50人以上」でキャンペーン配信、スコア式に「直近売上＋社員数＋拠点数」を組み込む。

---

## 2. 事業内容・歴史に関する情報

| カラム名 | 提案理由・活用シーン |
|----------|----------------------|
| **業種** / **業種1〜7** / **業種（大）（中）（小）（細）** / **業種-大・中・小・細** | 業種・業態によるターゲティングの基盤。業種別キャンペーン、業種×規模のマトリクスセグメント、トークスクリプトの「御社の〇〇業界では…」といった文面生成に不可欠。 |
| **営業種目** | 実際の取り扱い品目・サービス。業種コードより具体的で、製品・ソリューションに紐づけたメッセージやオファー設計に利用。 |
| **設立** / **創業** | 設立年・創業年による「企業年齢」セグメント。創業〜10年／老舗など、ライフステージに応じた訴求やスコアリングに利用。 |
| **説明** | 短い業種・事業説明。セグメント名のラベルや、メール・DMの冒頭文の自動挿入に利用。 |
| **概要** / **概況** / **企業概要** | より詳細な事業内容・強み・沿革。リードスコアの「コンテンツ理解」、営業のトークスクリプト・商談メモの下書き、パーソナライズ文面の素材として価値が高い。 |

**MAでの典型的な使い方**: 業種×設立年でセグメント、企業概要を要約してスコアやタグに反映、トークスクリプトに「御社の概要から〇〇という強みが読み取れ…」と組み込む。

## 3. 取引関係・外部ネットワークに関する情報

| カラム名 | 提案理由・活用シーン |
|----------|----------------------|
| **取引先** | 主要取引先・顧客名。アカウントベースマーケティング（ABM）の「既存顧客と取引先が近い企業」へのアプローチ、紹介・参照の文脈づくりに有用。 |
| **取引先銀行** | メインバンク情報。金融機関別セグメント、与信・安定性の補助指標。銀行経由の共同キャンペーンやリスト提供の検討材料。 |
| **仕入れ先** | 調達先・サプライヤー。業界構造の把握、B2Bで「同じサプライチェーン」への横展開、競合・業界マップの作成に利用。 |

**MAでの典型的な使い方**: 取引先・仕入れ先をタグやセグメント条件に使い「〇〇取引先と同系統」をターゲットにした配信、取引先銀行で金融機関連携施策の対象リストを設計。

## 4. その他、デジタルマーケティングに不可欠な基本情報

| カラム名 | 提案理由・活用シーン |
|----------|----------------------|
| **URL** | 企業サイト。デジタルタッチポイントの起点、サイト行動とCRMの紐づけ、ドメイン単位の重複排除・マッチングに必須。MAの「ランディング先」「リードの起点」として不可欠。 |
| **都道府県** / **住所** / **所在地** | 地域セグメント（エリア別キャンペーン、拠点エリアマッチ）。地域×業種×規模のマルチ軸ターゲティング、オフライン施策の対象選定。 |
| **郵便番号** / **会社郵便番号** / **会社住所** | 住所・都道府県と同様の地理セグメント。DM・訪問ルート設計、地域密着型オファーの配信対象に利用。 |
| **電話番号** / **電話番号(窓口)** | アウトバウンド・テレアポ、SMS／音声チャネル用の連絡先。MAで「チャネル」として管理し、配信可否・優先度の条件に利用。 |
| **法人番号** | 企業の一意識別子。他システム・公的データとの突合、重複排除、名寄せのキーとして必須。MAと他DBの同期・スコア連携の基盤。 |
| **上場** / **区分** | 上場・非上場・市場区分。投資家向け／成長段階別のセグメント、メッセージトーンやオファー内容の切り替えに利用。 |
| **取引種別** | 譲渡・譲受・提携などM&A文脈の区分。MAの「商流」別セグメント、案件種別に応じたコンテンツ・フロー設計に直結。 |
| **NDA** / **AD** / **SBフラグ** | NDA締結・AD契約・ストロングバイヤー等の進捗フラグ。エンゲージメント・読了度に基づくセグメント、配信抑制・優先配信の条件として運用。 |
| **ステータス** / **状態** | 未送信・送信済・読了等の状態。MAのワークフロー・ライフサイクルステージと連携し、次のアクション決定に利用。 |
| **リストID** / **会社ID** / **ID** | 元リスト・システム上の企業ID。データソースのトレサビリティ、重複・統合時の突合キー、MAと外部ツールの連携に利用。 |

**MAでの典型的な使い方**: URLでサイトとCRMを紐づけ、都道府県・住所でエリア配信、法人番号で名寄せ・スコア連携、取引種別・NDA/AD/SBで「どの商流・どのステップ」に配信するかを制御。

## まとめ：カテゴリ別 推奨追加カラム一覧

| カテゴリ | 推奨追加カラム |
|----------|----------------|
| **1. 業績・規模** | 直近売上, 直近利益, 直近決算年月, 売上規模（百万円）, 社員数 |
| **2. 事業内容・歴史** | 業種, 業種（大）（中）（小）（細）, 業種1〜7, 業種-大・中・小・細, 営業種目, 設立, 創業, 説明, 概要, 概況, 企業概要 |
| **3. 取引・ネットワーク** | 取引先, 取引先銀行, 仕入れ先 |
| **4. 基本・デジタル** | URL, 都道府県, 住所, 所在地, 郵便番号, 会社郵便番号, 会社住所, 電話番号, 電話番号(窓口), 法人番号, 上場, 区分, 取引種別, NDA, AD, SBフラグ, ステータス, 状態, リストID, 会社ID, ID |

**注意**: `Unnamed: 38〜46` はヘッダーなし列のため取り込み対象からは除外。`コメント`・`備考`・`担当者コメント` は運用ルール（個人情報・内部メモの扱い）を決めたうえで、MAでは「内部メモ」や「配信対象外フラグ」などに限定利用することを推奨。

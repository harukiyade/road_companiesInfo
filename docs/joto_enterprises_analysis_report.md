**目的**: 取引種別＝「譲渡企業」のレコードについて、データ形式を分析し、DBカラムへの最適マッピングを提案する。

**対象**: `fixed_csv_3` 配下の全CSV（`fixed_csv_3/later` 除外）  
**抽出条件**: ヘッダー「取引種別」の値が「譲渡企業」  
**方法**: Pythonスクリプト `scripts/extract_joto_enterprises.py` で抽出し、異なる5ファイルから計10件を取得。

## 2. 項目ごとのデータ形式分析レポート

### 2.1 売上・利益

| 項目 | 結果 |
|------|------|
| **ヘッダー名** | 抽出した全ファイルで「**直近売上**」「**直近利益**」が存在。「売上規模（百万円）」は本10件のファイルには出現せず。 |
| **単位** | セル内に単位表記はなし。ファイルが `unit_million` 配下のため、**百万円単位**と解釈するのが一貫している（例: 7870878 → 約78.7億円）。 |
| **形式** | **数値のみ**（カンマなし）。例: `7870878`, `700000`, `300000`, `0`, `869867`, `6324`. |
| **直近決算年月** | **表記ゆれあり**。（ア）`2022年8月1日` のような日本語日付 （イ）`44774`, `44166` のような **Excelシリアル日付**（同一企業でもファイルにより 1988年4月1日 vs 32234 のように混在）。 |

**結論**: 売上・利益は「直近売上」「直近利益」で**数値のみ・百万円単位**。決算年月は文字列のまま格納するか、シリアルは日付に変換する必要あり。

---

### 2.2 株主・役員

| 項目 | 結果 |
|------|------|
| **取締役** | **役職付き・カンマ区切り**で列挙。例: `（代会）安井隆司（取）小野謙一郎，髙野雅史，荻島健人（監）中島一奈`。略称は（取）（監）（代会）（専）（社長）等。**姓名は分離されずフルネーム**。空のレコードあり（藤工房、M's建材など）。 |
| **株主** | **「名前（比率）」をカンマ区切り**で列挙する形式が多数。例: `安井隆司（３９．５％），中島郁文（３０．１％），アテック持株会（２９％）`。比率は全角`％`あり。**姓名は分離されず**。 |
| **比率の有無** | 比率ありが基本。一方で `吉田孝則` のみ、`宮本厚司（６０株）`（株数）、`金義広(60株)，` のように**比率なし・株数のみ・末尾カンマ**などのゆれあり。 |

**結論**: 株主・役員はいずれも**1セル内のテキスト**。役職・比率は括弧付きで埋め込まれており、DBにはテキストとして格納するか、必要に応じて後処理でパースする形が現実的。

### 2.3 拠点数（オフィス・工場・店舗）

| 項目 | 結果 |
|------|------|
| **列の有無** | **オフィス数・工場数・店舗数が独立した3列**で存在。 |
| **形式** | **数値のみ**（単位なし）。例: `5`, `1`, `0`, `3`, `2`. |
| **0の表記** | すべて **`0`**（文字列のゼロ）。空欄ではない。 |

**結論**: 3列とも**数値のみ・0は "0"**。そのまま整数カラム（office_count, factory_count, store_count）に流し込める。

### 2.4 事業内容（概要・説明・営業種目）

| 項目 | 結果 |
|------|------|
| **説明** | **短い1行**の事業説明。例: 「自動車系を主とした機械設計や電子設計、及びソフトウェア開発を行う会社」「うどんやきしめんなどの手作り麺の製造・販売を行う会社」。文字数はおおむね **30〜60文字程度**。 |
| **概要** | **より長い段落**。業績・強み・沿革・課題を含む。例: 「システムエンジニアの人材派遣業者であり、近年においては業務請負比率が拡大基調。競争力が高く、盤石な財務体質も当社の強み。」**最も詳しい情報**はここにある。 |
| **営業種目** | 今回抽出した5ファイルのヘッダーには **「営業種目」列はなし**。業種1・業種2・業種3はあり（業種の階層・複数業種）。 |

**結論**: この10件に限れば、**概要が最も詳しい**。説明は短い要約、営業種目は本サンプルでは未使用。他ファイルに営業種目がある場合は「品目・サービス名」に近い内容と想定される。

## 3. DBカラムへのマッピング最終提案

実際のデータ形式（単位・表記ゆれ・どの列に何が入っているか）を踏まえた最終提案。

### 3.1 売上候補（revenue, latest_revenue, latest_sales）

| CSV項目 | 推奨DBカラム | 理由 |
|---------|----------------|------|
| **直近売上** | **latest_revenue** | 「直近」の売上であることが列名と一致。数値は百万円単位のため、DB側で単位を統一（例: 円に換算）する場合はインポート時に倍率を適用すること。 |
| 直近利益 | （候補にないが）**latest_profit** 等 | 同様に「直近」の利益として扱うのが自然。 |

- **revenue**: 単一時点でない売上や累計に使うなら、直近売上とは別用途。直近売上は **latest_revenue** に流し込むのが最適。
- **latest_sales**: 意味は latest_revenue とほぼ同義。既存スキーマで latest_revenue が標準なら **latest_revenue** に統一することを推奨。

### 3.2 設立候補（founded_year, founding_year, established, established_date）

| CSV項目 | 推奨DBカラム | 理由 |
|---------|----------------|------|
| **設立** | **established_date**（日付）および **founding_year**（年のみ） | データが「1988年4月1日」のような日付文字列と、Excelシリアル（32234 等）の混在。**日付として解釈できるものは established_date** に格納し、**年（4桁）は founding_year** に格納する二重格納が検索・集計に便利。 |
| | | シリアル値はインポート前に日付型に変換する処理を推奨。 |

- **founded_year** と **founding_year**: どちらか一方に統一するなら **founding_year**（年のみ）を推奨。日付は **established_date**。
- **established**: 文言が曖昧なため、日付用に **established_date**、年用に **founding_year** を明示する命名を推奨。

### 3.3 概要候補（overview, company_description, business_descriptions, summary）

| CSV項目 | 推奨DBカラム | 理由 |
|---------|----------------|------|
| **説明** | **overview** または **summary** | 短い1行要約。MA・一覧表示用の「一言説明」として **overview** が適切。summary が「短い要約」専用なら **summary** でも可。 |
| **概要** | **business_descriptions** | 長文で事業・業績・強み・沿革を含む。**事業内容の詳細**として **business_descriptions** に流し込むのが最適。 |
| | **company_description** | 会社説明全般を1本で持つ設計なら、概要を **company_description** に格納し、business_descriptions は「複数段落・リスト」用に残す運用も可。 |

**推奨方針**:  
- **説明 → overview**（短い要約・一覧用）。  
- **概要 → business_descriptions**（詳細な事業・財務・沿革）。  
- company_description は「概要の別名」として使うか、概要と同一内容を格納するかはスキーマの役割に合わせて選択。

## 4. まとめ

- **譲渡企業**は「直近売上」「直近利益」「直近決算年月」で**数値／日付またはシリアル**が混在。売上は **latest_revenue**、設立は **established_date + founding_year**、短い説明は **overview**、詳細は **business_descriptions** への流し込みを推奨。

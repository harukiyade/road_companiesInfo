# CSVヘッダーのパターン分析とグループ化

## 分析対象
- **fixed_csv_3** 配下の全CSV（121ファイル）
- ヘッダー構成でグループ化

---

## ヘッダーのグループ分け

### グループA：法人番号なし・会社郵便番号型（53ファイル）
**特徴**: 法人番号がなく、「会社郵便番号」「会社住所」を使用。業種は「業種-大/中/小/細」の4段階。営業種目・株主・取締役・概況あり。

**代表ヘッダー**:
```
会社名, 電話番号, 会社郵便番号, 会社住所, URL, 代表者名, 代表者郵便番号, 代表者住所, 代表者誕生日,
営業種目, 設立, 株主, 取締役, 概況, 業種-大, 業種-中, 業種-小, 業種-細
```

**該当ファイル例**: import_firstTime_10, 11, 12, 13, 101 等（53件）

---

### グループB：法人番号なし・創業設立型（24ファイル）
**特徴**: 法人番号なし。「創業」「設立」「株式保有率」「役員」あり。業種は「業種（大）（中）（小）（細）」。郵便番号・住所が2組（会社・代表者）。

**代表ヘッダー**:
```
会社名, 電話番号, 郵便番号, 住所, URL, 代表者, 郵便番号, 住所, 創業, 設立, 株式保有率, 役員, 概要,
業種（大）, 業種（中）, 業種（小）, 業種（細）
```

**該当ファイル例**: import_firstTime_23, 78, 79, 80, 102 等（24件）

---

### グループC：取引管理フル型（15ファイル）
**特徴**: 「法人番号」「会社ID」「リストID」「取引種別」「NDA」「AD」「状態」「備考」あり。業種1〜3、財務情報（資本金・直近売上・直近利益）充実。

**代表ヘッダー**:
```
会社名, 都道府県, 代表者名, 法人番号, 会社ID, リストID, 取引種別, 状態, NDA, AD, 備考, URL,
業種1, 業種2, 業種3, 郵便番号, 住所, 設立, 電話番号(窓口), 代表者郵便番号, 代表者住所, 代表者誕生日,
資本金, 上場, 直近決算年月, 直近売上, 直近利益, 説明, 概要, 仕入れ先, 取引先, 取引先銀行, 取締役, 株主,
社員数, オフィス数, 工場数, 店舗数
```

**該当ファイル例**: 10_20251224, 11_20251224, 16_20251224 等（15件）

---

### グループD：法人番号・ID・取引種別型（9ファイル）
**特徴**: グループCと似るが「会社ID」「リストID」の代わりに「ID」のみ。取引種別・SBフラグ・NDA・AD・ステータスあり。

**代表ヘッダー**:
```
会社名, 都道府県, 代表者名, 法人番号, ID, 取引種別, SBフラグ, NDA, AD, ステータス, 備考, URL,
業種1, 業種2, 業種3, 郵便番号, 住所, 設立, 電話番号(窓口), 代表者郵便番号, 代表者住所, 代表者誕生日,
資本金, 上場, 直近決算年月, 直近売上, 直近利益, 説明, 概要, 仕入れ先, 取引先, 取引先銀行, 取締役, 株主,
社員数, オフィス数, 工場数, 店舗数
```

**該当ファイル例**: import_firstTime_108, 119, 36, 43, 44 等（9件）

---

代表者郵便番号, 代表者住所, 代表者誕生日, 資本金, 上場, 直近決算年月, 直近売上, 直近利益, 説明, 概要,

### グループH：法人番号先頭型（3ファイル）
**特徴**: 法人番号が先頭列。以降はグループAと同様（会社郵便番号・会社住所・業種-大/中/小/細）。

**代表ヘッダー**:
```
法人番号, 会社名, 電話番号, 会社郵便番号, 会社住所, URL, 代表者名, 代表者郵便番号, 代表者住所, 代表者誕生日,
営業種目, 設立, 株主, 取締役, 概況, 業種-大, 業種-中, 業種-小, 業種-細
```

**該当ファイル例**: import_firstTime_1, 2, 103（3件）

---

### グループI：yuzuri簡易型（12ファイル）
**特徴**: 列数が少ない。企業名・所在地・業種・区分・売上規模（百万円）・企業概要。取引先・株主等なし。

**代表ヘッダー**:
```
企業名, 所在地, 代表者名, 業種, 区分, 売上規模（百万円）, 企業概要
```

**該当ファイル例**: yuzuri_1, 12, 14, 16, 19 等（12件）

---

### グループJ：業種（細）型（2ファイル）
**特徴**: 業種3の次が「業種（細）」で、業種4ではなく括弧表記。法人番号あり、ID・取引種別なし。

**代表ヘッダー**:
```
会社名, 都道府県, 代表者名, 法人番号, URL, 業種1, 業種2, 業種3, 業種（細）, 郵便番号, 住所, 設立, ...
```

**該当ファイル例**: import_firstTime_107, 110（2件）

## グループごとのDBカラム対応（主要マッピング）

| CSVヘッダー | DBカラム |
|-------------|----------|
| 会社名 / 企業名 | name |
| 法人番号 | corporate_number |
| 電話番号 / 電話番号(窓口) | phone_number |
| 都道府県 | prefecture |
| 住所 / 会社住所 / 所在地 | address |
| 郵便番号 / 会社郵便番号 | postal_code |
| 代表者名 / 代表者 | representative_name |
| 代表者郵便番号 | representative_postal_code |
| 代表者住所 | representative_home_address |
| 代表者誕生日 | representative_birth_date |
| URL / 企業ホームページURL | company_url |
| 業種1 / 業種-大 / 業種（大） | industry_large |
| 業種2 / 業種-中 / 業種（中） | industry_middle |
| 業種3 / 業種-小 / 業種（小） | industry_small |
| 業種4 / 業種-細 / 業種（細） | industry_detail |
| 資本金 | capital_stock |
| 直近売上 | latest_revenue |
| 直近利益 | latest_profit |
| 設立 / 設立年月日 | established |
| 社員数 / 従業員数 | employee_count |
| 説明 / 概要 / 概況 / 企業概要 | overview |
| 取引種別 | transaction_type |
| 仕入れ先 | suppliers |
| 取引先 | clients |
| 取締役 | executives |
| 株主 | shareholders |
